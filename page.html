Git vs GitHub

Git

Git é um sistema de controle de versão distribuído amplamente utilizado por desenvolvedores de software.

Principais comandos do Git:

git init: Inicializa um repositório Git em um diretório.
git add: Adiciona arquivos ao índice.
git commit -m "Mensagem de commit": Grava as alterações no repositório com uma mensagem de commit.
git status: Mostra o status atual do repositório, incluindo arquivos modificados, adicionados e não rastreados.
git log: Mostra o histórico de commits do repositório.
git push origin master: Envia as alterações locais para o branch master de um repositório remoto chamado "origin".
git pull origin master: Puxa as alterações do branch master de um repositório remoto chamado "origin" para o repositório local.
git branch: Lista todos os branches no repositório e mostra o branch atual com um asterisco.
git checkout -b novo-branch: Cria um novo branch chamado "novo-branch" e muda para ele.
git merge outro-branch: Combina as alterações do branch "outro-branch" com o branch atual.
git remote add origin URL: Adiciona um repositório remoto chamado "origin" com a URL especificada.
git clone URL: Clona um repositório remoto para o diretório local.

GitHub

GitHub é uma plataforma de hospedagem de código-fonte baseada em Git. Além de hospedar repositórios, oferece recursos colaborativos e de gerenciamento.

Principais recursos do GitHub:

  • Hospedagem de repositórios Git.
  • Colaboração através de pull requests.
  • Controle de versões e histórico de alterações.
  • Gerenciamento de problemas e tarefas.
  • Integração com ferramentas de CI/CD.
  • Revisão de código e discussão.
  • Wikis e documentação de projetos.
  • GitHub Actions para automação de fluxos de trabalho.

Exemplos Práticos

Aqui estão alguns exemplos práticos dos comandos do Git em ação:

Exemplo 1: Como inicializar um repositório Git:

git init

Este comando cria um novo repositório Git no diretório atual.

Exemplo 2: Como adicionar arquivos ao índice:

git add index.html

Este comando adiciona o arquivo "index.html" ao índice para prepará-lo para um commit.

Exemplo 3: Como gravar alterações no repositório:

git commit -m "Mensagem do commit"

Este comando grava as alterações adicionadas ao índice no repositório com uma mensagem de commit.

Exemplo 4: Como verificar o status do repositório:

git status

Este comando mostra o status atual do repositório, incluindo arquivos modificados, adicionados e não rastreados.

Exemplo 5: Como visualizar o histórico de commits:

git log

Este comando exibe o histórico de commits do repositório, mostrando informações como autor, data e mensagem de cada commit.

Exemplo 6: Como criar um novo branch:

git branch nome-do-branch

Este comando cria um novo branch com o nome especificado.

Exemplo 7: Como mudar para outro branch:

git checkout nome-do-branch

Este comando muda para o branch especificado.

Exemplo 8: Como combinar alterações de um branch para outro:

git merge nome-do-branch

Este comando combina as alterações do branch especificado com o branch atual.

Exemplo 9: Como remover um arquivo do repositório:

git rm arquivo.txt

Este comando remove o arquivo especificado do repositório e do diretório de trabalho.

Exemplo 10: Como desfazer as alterações em um arquivo não commitadas:

git checkout -- arquivo.txt

Este comando descarta as alterações não commitadas no arquivo especificado.

Conclusão

O Git e o GitHub são ferramentas essenciais para o desenvolvimento de software colaborativo. Dominar os comandos do Git e aproveitar os recursos do GitHub podem aumentar significativamente a eficiência e a qualidade do seu trabalho.

Continue explorando e praticando para aprimorar suas habilidades!
